Defect: "Switch Scripture View" cannot toggle back from Markers to Formatted in simple interface mode.
File: extensions/src/platform-scripture-editor/src/platform-scripture-editor.web-view.tsx:739

ROOT CAUSE
The changeScriptureView handler decided the next view from the resolved markerMode:
    setViewType(viewOptions.markerMode === 'hidden' ? 'markers' : 'formatted');
where viewOptions = getViewOptionsForType(viewType, isPowerMode).

In SIMPLE interface mode (platform.interfaceMode default 'simple'), both view types resolve to
markerMode 'hidden':
  - formatted -> getViewOptions('paragraph-structure')                 -> markerMode 'hidden'
  - markers   -> { ...getViewOptions('paragraph-structure'), noteMode } -> markerMode 'hidden'
So once viewType === 'markers', the rule computes 'hidden' === 'hidden' -> 'markers' again: the
user is stuck in the read-only markers view. viewType is persisted via useWebViewState, so a reload
does not recover. line 739 is the ONLY caller of setViewType. Power mode works because markers
resolves to markerMode 'visible'.

Verified library values (.yalc/@eten-tech-foundation/platform-editor/dist/index.js):
  default view mode = "formatted" -> markerMode "hidden"
  "paragraph-structure"           -> markerMode "hidden"
  "unformatted"                   -> markerMode "editable"

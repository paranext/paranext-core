# Rebase platform-bible-react branch onto origin/main

Rebase the current branch onto `origin/main`, resolving any conflicts inside `dist/` directories by discarding the conflicting dist changes (they will be regenerated by the build), then rebuild `platform-bible-react`, commit, and push.

## Steps

1. **Confirm working tree is clean.**

   ```bash
   git status --porcelain
   ```

   If there are uncommitted changes, stop and ask the user how to proceed. Do not stash silently.

2. **Fetch and record the starting branch.**

   ```bash
   git fetch origin main
   BRANCH=$(git rev-parse --abbrev-ref HEAD)
   ```

   If `BRANCH` is `main`, stop — this command is only for rebasing feature branches.

3. **Start the rebase.**

   ```bash
   git rebase origin/main
   ```

4. **Resolve conflicts in a loop until the rebase completes.**

   While `git status` reports a rebase in progress:

   - Identify conflicted paths: `git diff --name-only --diff-filter=U`
   - For every conflicted path **inside a `dist/` directory** (any depth — e.g. `lib/platform-bible-react/dist/...`, `lib/platform-bible-utils/dist/...`, `extensions/**/dist/...`):
     - Take the incoming (main) version and stage it: `git checkout --ours -- <path> && git add <path>`
       - (During rebase, `--ours` refers to the branch being rebased onto, i.e. `origin/main`. We are discarding the feature branch's dist changes because the build will regenerate them.)
   - If, after handling all `dist/` conflicts, **non-dist conflicts remain**, stop and surface them to the user with a short summary of each file. Do not guess resolutions.
   - Once all conflicts for the current step are resolved and staged, continue:

     ```bash
     git rebase --continue
     ```

   Repeat until the rebase reports success (no rebase in progress).

   If at any point you need to abort, use `git rebase --abort` and report to the user — do not leave the tree in a half-rebased state.

5. **Rebuild `platform-bible-react`.**

   ```bash
   cd lib/platform-bible-react && npm run build
   cd ../..
   ```

   If the build fails, stop and surface the error. Do not commit a broken build.

6. **Commit the regenerated dist (if there are changes).**

   ```bash
   git status --porcelain
   ```

   If `lib/platform-bible-react/dist/` (or any other regenerated artifacts) shows changes, stage and commit them:

   ```bash
   git add lib/platform-bible-react/dist
   git commit -m "chore: rebuild platform-bible-react dist after rebase onto main"
   ```

   If there are no changes, skip this step.

7. **Force-push with lease.**

   Because the branch was rebased, a normal push will be rejected. Use `--force-with-lease` (safer than `--force`) to avoid clobbering upstream work you don't know about:

   ```bash
   git push --force-with-lease origin "$BRANCH"
   ```

   If the push is rejected because the lease check failed, stop and surface it — do not escalate to `--force`.

8. **Report.**

   Summarize:
   - How many conflicts were resolved and how many were in `dist/` vs elsewhere.
   - Whether a dist-rebuild commit was created.
   - The final `HEAD` SHA and the branch pushed.
